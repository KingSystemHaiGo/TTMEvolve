# TTMEvolve v0.4.2 Onboarding Closure

Date: 2026-06-24

## Stable Scope

This small release closes the v0.4.1 diagnostic work into a single onboarding path for humans and arbitrary LLMs. It does not claim that live Maker/API validation is finished; it makes the next validation pass fast, evidence-driven, and copyable.

## What Changed

- Added `GET /agent/onboarding?session_id=...&steps=20` and `/agent/onboarding.md`.
- Runtime Contract now exposes `communication.onboarding_bundle` and makes it the first external-agent attach step.
- The Onboarding Bundle combines Runtime Readiness, Evidence Bundle, Runtime Advice, MakerMCP authority, layer summary, runtime metrics, learning status, API call proof, token strategy, endpoint map, and closure gate checks.
- AgentWorkbench External Agent Boot now prioritizes Onboarding and has `Copy Onboarding`.
- The closure gate maps the six product requirements into compact checks: any-LLM startup, MakerMCP binding, layer independence, token efficiency, frontend/backend loop, theme/release readiness.
- The bundle carries TapTap Maker Plus reference principles: do not bypass MakerMCP, prefer schema/evidence-driven surfaces, and expose raw errors/logs only when compact summaries are insufficient.

## Stable Definition

- Normal GUI execution must use a configured API provider or explicit local model; no mock fallback.
- Any attached LLM can start from the Onboarding Bundle before selecting tools.
- MakerMCP authority and guard evidence are visible before Maker side effects.
- Agent, Core Runtime, and Learning evidence are independently pullable and summarized.
- Token strategy starts from onboarding/readiness/evidence/context summaries before raw histories.
- Workbench and backend expose the same compact evidence path.

## Known Live Validation Gaps

- Real provider Probe with the user's API key.
- Real MakerMCP-connected task proving remote authority, guard decision, and post-action evidence.
- Runtime metrics from a live Maker task, including token/cache/tool-ranking evidence.
- Background Learning completion during a live run.
- Electron visual smoke for BrowserView preview/chat usability.
