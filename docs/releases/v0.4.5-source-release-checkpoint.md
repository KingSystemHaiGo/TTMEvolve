# v0.4.5 Source Release Checkpoint

Date: 2026-06-27 12:57

## Purpose

This checkpoint creates a safe source release package for the current verified TTMEvolve state. It is not a full offline runtime bundle and does not claim installer signing, embedded runtime completeness, or production embedding semantic quality.

## What Changed

- `scripts/package_release.py` now builds a source checkpoint package under `release-artifacts/` by default.
- The packager excludes local runtime/private/build state:
  - `config.json`, `.env*`, `.mcp.json`
  - `storage/`, `portable/`, `vendor/`, `models/`, `workspace/`
  - `node_modules/`, frontend/electron `dist/`, `src-tauri/target/`, `logs/`
  - local agent/editor state such as `.codex/`, `.cursor/`, `.claude/`, `.agents/`
- The packager writes a manifest next to the zip with file count, size, SHA-256, excluded prefixes, and forbidden-entry findings.
- The packager validates archive entries before and after writing the zip, and fails if forbidden paths are present.
- `.gitignore` now excludes `release-artifacts/`.
- `scripts/release_readiness.py` now audits the source package, manifest, visible launch surface, git-ignore state, and explicit claim levels through `--mode source-checkpoint` and `--mode full-offline`.
