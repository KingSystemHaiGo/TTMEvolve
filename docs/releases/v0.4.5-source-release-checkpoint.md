# v0.4.5 源码 checkpoint / v0.4.5 Source Release Checkpoint

日期 / Date: 2026-06-27

## 目的 / Purpose

这个 checkpoint 为当前已验证的 TTMEvolve 状态创建安全源码包。它不是完整离线产品发布，也不声明签名安装包、Maker 远程构建 smoke 或生产 embedding 语义质量。

This checkpoint creates a safe source release package for the current verified TTMEvolve state. It is not a full offline product release and does not claim installer signing, Maker remote build smoke, or production embedding semantic quality.

## 变更 / What Changed

- `scripts/package_release.py` 默认在 `release-artifacts/` 下构建 source checkpoint package。
- 打包器排除本地运行时、私有配置和构建状态：`config.json`、`.env*`、`.mcp.json`、`storage/`、`portable/`、`vendor/`、`models/`、`workspace/`、`node_modules/`、`src-tauri/target/`、`logs/`。
- 打包器会在 zip 旁写入 manifest，包含 file count、size、SHA-256、excluded prefixes 和 forbidden-entry findings。
- 打包器在写入前后都会校验 archive entries，发现 forbidden paths 就失败。
- `.gitignore` 排除 `release-artifacts/`。
- `scripts/release_readiness.py` 通过 `--mode source-checkpoint` 和 `--mode full-offline` 审计源码包、manifest、可见启动入口、git-ignore 状态和 claim levels。

- `scripts/package_release.py` now builds a source checkpoint package under `release-artifacts/` by default.
- The packager excludes local runtime, private config, and build state: `config.json`, `.env*`, `.mcp.json`, `storage/`, `portable/`, `vendor/`, `models/`, `workspace/`, `node_modules/`, `src-tauri/target/`, and `logs/`.
- The packager writes a manifest next to the zip with file count, size, SHA-256, excluded prefixes, and forbidden-entry findings.
- The packager validates archive entries before and after writing the zip, and fails if forbidden paths are present.
- `.gitignore` excludes `release-artifacts/`.
- `scripts/release_readiness.py` audits the source package, manifest, visible launch surface, git-ignore state, and explicit claim levels through `--mode source-checkpoint` and `--mode full-offline`.
