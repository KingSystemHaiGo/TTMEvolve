# v0.4.4 Portable Agent Root

Date: 2026-06-24

## Decision

`v0.4.4-portable-agent-root` is a stable small version for the "TTMEvolve as a self-contained Agent folder" requirement.

This release does not yet reorganize the messy development repository into a clean distributable package. It first makes runtime path ownership explicit, observable, and test-covered so the app can stop silently writing caches, temp files, browser state, Maker auth, npm/npx state, and model caches into `C:\Users\...` by default.

## What Changed

- Added `core/portable_env.py`.
  - Applies repo-local `portable/` paths for home, cache, temp, Maker auth, pip, npm, HuggingFace, sentence-transformers, torch, matplotlib, and Playwright.
  - Exposes `portable_diagnostics()` for no-network path health checks.
- Added `GET /runtime/portable`.
  - Reports effective portable env paths, config-relative roots, unset variables, outside-project paths, and `C:\Users\...` leakage risks.
  - Runtime Contract, Evidence, Onboarding, Quickstart, and Handoff now include `portable_runtime`.
- Patched startup surfaces.
  - `main.py`, `server/electron_entry.py`, `start.ps1`, `start-gui.ps1`, and embedded setup apply portable env early.
  - `scripts/bootstrap.py` passes portable env into pip, wheel, Playwright, and offline-env subprocesses.
  - Electron app data/cache/session paths are redirected under `portable/electron`.
- Patched config resolution.
  - `project_root`, `storage_root`, `runtime.portable_root`, `llm.model_path`, and `maker_mcp.cwd` resolve relative to the config file directory.
  - Default Maker project root is now `./workspace/default-maker-project`, keeping game projects separate from the TTMEvolve app root.
- Patched Maker auth and browser fallback.
  - Maker auth reads `TTM_MAKER_HOME` instead of hard-coding `Path.home()/.taptap-maker`.
  - Playwright browser downloads fall back to `portable/cache/playwright` when `vendor/playwright` is absent.

## New Endpoint

```text
GET /runtime/portable
```

Use this before real Maker development testing. `status=ready` means the managed runtime paths are set and under the Agent root. `blocked` means a required portable variable is missing or leaking into a Windows user directory.
