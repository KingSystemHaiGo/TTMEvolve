# TTMEvolve v0.7.0 — 完整 Windows 桌面应用

> **发布日期**：2026-06-26
> **版本号**：v0.6.0 → v0.7.0（重大版本）
> **核心变更**：从 Electron 迁移到 Tauri 2.x + 全面转向云端 LLM

---

## 🎉 重大变更

### 1. 架构升级：Tauri 2.x

- **桌面壳**：Rust 内核 + WebView2
- **后端**：Python（FastAPI）不变
- **Electron**：标记为 deprecated（v0.7.1），v0.8.0 删除

### 2. LLM 策略：全面云端

- **删除**：llama-cpp-python、torch、sentence-transformers、huggingface_hub
- **新增**：LLMRouter 多 Provider 路由 + 故障转移
- **支持**：OpenAI、Anthropic Claude、DeepSeek、Qwen、Zhipu、Moonshot、Doubao、SiliconFlow、MiniMax

### 3. 包体积优化

| 配置 | v0.6.0 | v0.7.0 |
|------|--------|--------|
| 极简（云端） | ~450MB | **~200MB** |
| 含本地模型 | ~4.5GB | **不打包** |

## ✨ 新增功能

### 主题系统

- 亮色 / 暗色双主题
- 完整设计令牌（颜色、圆角、阴影、间距）
- `localStorage` 持久化 + 系统偏好检测
- `frontend/src/styles/theme.css`
- `frontend/src/hooks/useTheme.ts`

### Settings 页面（5 面板）

| 面板 | 内容 |
|------|------|
| 当前项目 | 项目名 / 路径 / project_id / config.json |
| MCP Runtime | 状态 / processId / cwd / tools/list 时间 |
| MCP Schema | 工具总数 / 分类 / 表单来源 |
| 本地工作台能力 | 内嵌 Python / Node / Maker MCP / HTTP API |
| 开发者模式 | F12 开关 / 日志 / DevTools |

### UI 组件库

- `Button` — 5 变体 × 3 尺寸
- `Panel` — 卡片容器
- `Toggle` — 开关组件
- `SettingRow` — key/value 行

### 后端 API 端点

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/settings/runtime-info` | 项目 / MCP / schema / portable / LLM Router |
| GET | `/api/settings/llm-providers` | Provider presets + router stats |
| POST | `/api/settings/devtools` | 开发者模式操作 |

### Tauri 命令（前端 → Rust）

| 命令 | 用途 |
|------|------|
| `server_status` | 后端状态 |
| `server_start` | 启动 Python 后端 |
| `server_stop` | 停止后端 |
| `open_devtools` | 打开 DevTools |

### 内嵌环境构建（portable/）

```
scripts/build-portable/
├── config.json           ← 下载源 + 依赖列表
├── build_python.py       ← Python 嵌入式
├── build_node.py         ← Node.js 嵌入式
├── build_site_packages.py← pip install 精简依赖
├── build_all.py          ← 一键编排
└── verify_portable.py    ← 5 项离线验证
```

### 性能监控

- `core/perf_monitor.py` — 轻量计时器
- p50 / p95 / max / count 统计
- 可选启用，零开销

## 📦 安装与运行

### Windows（生产）

```cmd
# 解压 TTMEvolve-0.7.0-portable.zip
# 双击 start-tauri.bat 即可
```

### Windows（开发者）

```cmd
:: 1. 启动 Tauri dev 模式
cd src-tauri
cargo tauri dev
```

### 跨平台

```bash
# Linux / macOS
./start-tauri.sh
```

### 启动优先级（portable 优先）

1. `portable/python/python.exe`（内嵌）
2. `.venv/Scripts/python.exe`（虚拟环境）
3. 系统 PATH 上的 `python`

## 🔧 升级指南

### 从 v0.6.x 升级

1. 拉取最新代码
2. 安装新依赖（如果需要）：
   ```bash
   .venv/Scripts/pip install -r requirements.txt
   ```
3. 配置 LLM Provider（config.json 或 Settings 页）：
   ```json
   {
     "llm": {
       "provider": "anthropic",
       "api_key": "${ANTHROPIC_API_KEY}",
       "fallback_providers": [
         {"provider": "deepseek", "model": "deepseek-chat"}
       ]
     }
   }
   ```
4. 启动新版本：
   ```bash
   ./start-tauri.sh
   ```

## 📋 已知问题

1. **首次 cargo build 慢**：Rust 编译依赖较多
2. **WebView2 仅 Windows**：Linux/macOS 用 webkit2gtk / webkit
3. **图标占位**：需要品牌图标替换
4. **Tauri 端到端测试**：当前仅单元测试，无 e2e（v0.7.1 补）
