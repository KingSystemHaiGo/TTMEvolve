# v0.4.5 One-Click Practice Entry

Date: 2026-06-24

## Purpose

Real Maker testing should not require the user to manually assemble install/init/config/GUI commands.

This checkpoint adds visible Windows GUI launchers. The command scripts are backend/bootstrap implementation details, not the user-facing entry.

Primary visible launchers:

```text
TTMEvolve.vbs
TTMEvolve-Practice.vbs
```

Optional Windows shortcuts can be generated with:

```powershell
.\scripts\create_windows_shortcuts.ps1
```

It also moves the same practice flow into the GUI so the command window is no longer the primary interaction surface.

## What It Does

- Applies the portable runtime environment.
- Creates or reuses `workspace/smoke-maker-game`.
- Refuses to use the TTMEvolve app root as the Maker game directory.
- Writes `config.json` so:
  - `project_root` points at the separate Maker game directory.
  - `maker_mcp.cwd` points at the same Maker game directory.
  - `storage_root` remains `./storage`.
  - `runtime.portable_root` remains `./portable`.
- Runs Maker MCP install:

```powershell
npx -y @taptap/maker install --ide codex,cursor,claude
```

- Runs Maker project init from the selected game directory when it is not already initialized:

```powershell
npx -y @taptap/maker init
```

- Launches `start-gui.ps1`.
- Prints the first diagnostic URLs:
  - `/runtime/portable`
  - `/maker/setup-status`
  - `/maker/tool-audit`
  - `/runtime/readiness`

## GUI Practice Wizard

The Maker Setup strip now has a `Practice` button. It calls backend practice endpoints and keeps the flow inside TTMEvolve:

- `POST /maker/practice/start`
- `GET /maker/practice/status`
- `POST /maker/practice/input`
- `POST /maker/practice/cancel`

The GUI shows:

- current setup step
- selected Maker project directory
- recent Maker CLI output
- auth URL action when Maker prints a PAT/login URL
- input box when Maker CLI asks for app index, `0`/new, or `all`

This means the expected real-practice path is now:

1. Open TTMEvolve GUI.
2. Click `Practice` in the Maker Setup strip.
3. If auth is required, click `Open Auth` and finish TapTap authorization in the embedded Maker browser.
4. If Maker asks for an app/project choice, answer in the GUI input box.
5. Start the tiny Maker smoke task in chat.

## Useful Variants

```powershell
.\start-practice.bat -ProjectName my-first-maker-game
.\start-practice.bat -ProjectDir D:\CC\MyMakerGame
.\start-practice.bat -SkipMakerInstall
.\start-practice.bat -SkipMakerInit
.\start-practice.bat -NoGui
```

These variants are for development/support. Normal users should start from the visible GUI launcher or shortcut.

## Boundary

TapTap account authorization may still require human interaction if Maker CLI requires login. The script keeps auth state under `portable/home/.taptap-maker`, but it cannot safely bypass TapTap authorization.
