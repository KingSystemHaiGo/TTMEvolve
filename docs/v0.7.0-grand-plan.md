## 0. 架构决策（已确认）

### 设计原则（用户硬要求）

> "我们的设计目的上就是要不安装任何环境依赖就能直接使用，所以内嵌环境也是都要推到 git 的"
> "发版的时候，要带有内嵌环境，运行优先使用内嵌环境"

**硬约束：**

| # | 约束 | 落地 |
|---|------|------|
| 1 | **不装任何环境依赖即可运行** | 内嵌 Python + Node + 必要运行时打包 |
| 2 | **内嵌环境推到 git** | `portable/` 目录完整提交（含二进制） |
| 3 | **运行优先内嵌环境** | start.sh / start.bat 优先 portable |
| 4 | **离线可用** | 不依赖 npm install / pip install / 模型下载 |

---

### 多方案对比

| 架构 | 性能 | 响应 | 后台占用 | 开发 | 升级 |
|------|------|------|---------|------|------|
| Electron（现状） | ❌ 重 | ❌ 慢 | ❌ 高 (~150MB) | ✅ | ✅ |
| Tauri 2.x + Rust | ✅ 轻 | ✅ 快 | ✅ 低 (~10MB) | ⚠️ 中 | ✅ |
| Tauri + Rust 内核（pyo3） | ✅✅ 极佳 | ✅✅ 极快 | ✅✅ 极低 | ❌ 难 | ⚠️ |
| Flutter Desktop | ✅ | ✅ | ✅ 低 | ⚠️ 中 | ⚠️ |
| WebView2 直嵌 | ✅✅ | ✅✅ | ✅✅ | ✅ | ✅ |
| **混合：Tauri + Rust 关键路径** | ✅✅ | ✅✅ | ✅✅ | ⚠️ 中 | ✅ |

### 4.2 强化目标

| 项 | 目标 |
|----|------|
| start.bat | **优先 portable/python + portable/node**，找不到才用系统 |
| .venv 打包 | 推到 git 或首次自动解压 |
| 模型打包 | `models/*.gguf` 已推 git |
| 离线启动 | 断网测试通过 |
| 体积 | 优化目标：完整包 < 300MB |

### 4.3 start.sh / start.bat 改造

```bash
#!/usr/bin/env bash
# start.sh — 优先内嵌环境
set -e

# 1. 内嵌 Python
if [ -x "./portable/python/bin/python3" ]; then
    PYTHON="./portable/python/bin/python3"
elif [ -x "./.venv/Scripts/python.exe" ]; then
    PYTHON="./.venv/Scripts/python.exe"
elif command -v python3 >/dev/null 2>&1; then
    PYTHON="python3"
else
    echo "❌ 未找到 Python，请运行 scripts/setup-portable.sh"
    exit 1
fi

# 2. 内嵌 Node
if [ -x "./portable/node/bin/node" ]; then
    NODE="./portable/node/bin/node"
elif command -v node >/dev/null 2>&1; then
    NODE="node"
else
    echo "❌ 未找到 Node"
    exit 1
fi

# 3. 启动后端
exec "$PYTHON" main.py --embedded
```
