## 2. Bug 列表

### Bug #1: `/api/settings/*` 端点返回 404

**现象：**
```bash
$ curl http://127.0.0.1:7345/api/settings/runtime-info
HTTP/1.0 404 Not found
```

**对比：**
```bash
$ curl http://127.0.0.1:7345/health     # 200 OK
$ curl http://127.0.0.1:7345/config    # 200 OK
$ curl http://127.0.0.1:7345/llm/providers  # 200 OK
$ curl http://127.0.0.1:7345/api/settings/runtime-info  # 404
```

**已查：**
- `server/app_server.py:2542` 注册了 `/api/settings/runtime-info` 路由
- `do_GET` 函数正确返回 `build_settings_runtime_info(server)` 的结果
- `build_settings_runtime_info` 在单元测试中工作正常

**可能原因：**
- `build_settings_runtime_info(server)` 在 HTTP 上下文中抛异常
- 异常被 `do_GET` 上层某处 catch 后转换为 404
- 需要更多日志调试（已加 print 但被 `log_message` 静默）

### Bug #2: `main.py --serve` 不支持 `--port` 参数

**现象：**
```bash
$ python main.py --serve --port 8765
usage: main.py [-h] [--config CONFIG] ...
main.py: error: unrecognized arguments: --port
```

**修复方向：**
- `argparse` 添加 `--port` 参数（默认 7345）

### Bug #3: `--mock` 选项可能未生效

**现象：**
```bash
$ python main.py --serve --mock
$ curl /health
{"provider": "minimax", "llm_class": "MiniMaxLLM", ...}  # 不是 mock
```

**可能原因：**
- main.py 加载 config 后才解析 `--mock`，但 `create_default_app_server` 在解析前就创建了 agent

### Bug #4: Windows GBK 编码错误

**现象：**
```python
exec(open('main.py').read())  # 调试时
UnicodeDecodeError: 'gbk' codec can't decode byte 0xab
```

**说明：**
- main.py 在 Windows 下读取时遇到非 GBK 字符
- 这只影响调试代码，运行时不会触发
- 但需要加 `# -*- coding: utf-8 -*-` 头文件声明
