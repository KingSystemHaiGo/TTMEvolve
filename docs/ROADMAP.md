# 路线图 / Roadmap

TTMEvolve 当前围绕稳定 source checkpoint 和明确 claim gates 组织开发。

TTMEvolve is currently organized around a stable source checkpoint and explicit claim gates.

## 当前状态 / Current

- Tauri + React 桌面外壳。
- Python App Server，包含 session APIs 和 evidence endpoints。
- Maker setup/status/tool audit 支持。
- API-first LLM provider routing，并带 probe evidence。
- Source package 和 readiness audits。

- Tauri + React desktop shell.
- Python App Server with session APIs and evidence endpoints.
- Maker setup/status/tool audit support.
- API-first LLM provider routing with probe evidence.
- Source package and readiness audits.

## 下一批发布 gate / Next Release Gates

- 签名安装包。
- Maker 远程构建 side-effect smoke。
- 使用真实 golden corpus 和 production embedding artifact 的生产 RAG 语义质量证明。
- 绑定 GitHub Releases 的更完整公开 release notes。

- Signed installer artifacts.
- Maker remote build side-effect smoke.
- Production RAG semantic-quality proof using a real golden corpus and production embedding artifact.
- Better public release notes tied to GitHub Releases.

## 当前 checkpoint 的非目标 / Non-Goals For The Current Checkpoint

- 不声明完整离线商业发布 ready。
- 不在缺少评估证据时声明生产语义 RAG 质量。
- 不发布内部项目记忆或私有运行时状态。

- Claiming full offline commercial release readiness.
- Claiming production semantic RAG quality without evaluation evidence.
- Publishing internal project memory or private runtime state.
