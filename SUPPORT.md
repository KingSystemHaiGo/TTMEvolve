# 支持 / Support

可复现 bug、文档缺口和功能请求请使用 GitHub Issues。

Use GitHub Issues for reproducible bugs, documentation gaps, and feature requests.

## 提 issue 前 / Before Opening an Issue

请尽量包含：

- 操作系统和 shell
- TTMEvolve commit 或 release tag
- 使用的启动方式，例如 `start-tauri.bat`
- 相关命令输出
- 适用时附上 `/health`、`/runtime/readiness` 或 `/maker/setup-status` 证据

Before opening an issue, include:

- OS and shell
- TTMEvolve commit or release tag
- startup path used, such as `start-tauri.bat`
- relevant command output
- `/health`, `/runtime/readiness`, or `/maker/setup-status` evidence when applicable

不要在公开 issue 中粘贴 API keys、Maker auth state、私有项目素材或本地模型文件。

Do not paste API keys, Maker auth state, private project assets, or local model files into public issues.
